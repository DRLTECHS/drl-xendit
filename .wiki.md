# Project Summary
The project is a web application that provides professional software development services, including website and mobile app development, and e-commerce solutions. It features an admin UI for managing payment API keys and integrates with Xendit for payment processing.

# Project Module Description
The project consists of multiple functional modules, including:
- **Frontend**: User interface components, pages, and API interactions.
- **Backend**: API routes, database models, and service integrations.
- **Admin Dashboard**: Management of payment settings and user authentication.

# Directory Tree
```
app/
├── backend/
│   ├── core/                     # Core functionalities like authentication and database
│   ├── data_models/              # JSON files for data models
│   ├── dependencies/              # Dependency management
│   ├── middlewares/              # Middleware for request handling
│   ├── routers/                  # API route definitions
│   ├── services/                 # Business logic and service interactions
│   ├── main.py                   # Entry point for the backend
├── frontend/
│   ├── components/               # Reusable UI components
│   ├── pages/                    # Application pages
│   ├── public/                   # Public assets like images and favicon
│   ├── src/                      # Source files for the frontend application
│   ├── index.html                # Main HTML file
│   └── package.json              # Frontend dependencies and scripts
└── start_app_v2.sh              # Script to start the application
```

# File Description Inventory
- **app/backend/**: Contains backend logic, including routes, models, and services.
- **app/frontend/**: Contains frontend components, pages, and assets for the web application.
- **app/start_app_v2.sh**: Shell script to start the application.

# Technology Stack
- **Frontend**: React, TypeScript, Tailwind CSS
- **Backend**: Python, FastAPI
- **Database**: JSON files for data models
- **Payment Integration**: Xendit API

# Usage
1. **Install Dependencies**:
   - Navigate to the frontend directory and run the installation command.
2. **Build the Application**:
   - Execute the build command to compile the frontend.
3. **Run the Application**:
   - Use the provided script to start the application.
